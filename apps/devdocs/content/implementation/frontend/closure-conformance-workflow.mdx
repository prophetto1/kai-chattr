## What The Current Hook Does Exactly

`closure-gate.mjs`:

1. Loads config using `closure-gate-core.mjs`.
2. Derives its hook id from filename: `closure-gate`.
3. Allows immediately if disabled.
4. Allows if no `settings.active.name`.
5. Allows if tool is not in `editTools`.
6. Reads only `tool_input.file_path`.
7. If `targetGlob` is set, uses substring matching, not glob matching.
8. Looks for `manifests/<name>/<name>.source-manifest.json`.
9. Denies if the manifest is missing.
10. Allows if the manifest is unreadable, fail-open.
11. Denies if any module verdict is not `port`, `adapter`, or `cut`.
12. Allows once all verdicts are set.

Main bug: for Codex `apply_patch`, it does not parse patch paths. So path scoping is not reliable.

## What Needs To Go Into `config.json`

Add a **new** hook entry, not the old `closure-gate` entry:

```json
{
  "id": "depcruiser-migrations",
  "name": "Depcruiser Migrations Gate",
  "category": "gate",
  "event": "PreToolUse",
  "match": { "tools": ["Edit", "Write", "MultiEdit", "NotebookEdit", "apply_patch"] },
  "script": { "path": "depcruiser-migrations/depcruiser-migrations.mjs", "runtime": "node" },
  "scope": { "projects": ["*"], "paths": ["**"] },
  "enabled": false,
  "failPolicy": "open",
  "settings": {
    "stateDir": "E:/hooks/.state/depcruiser-migrations",
    "manifestsDir": "E:/hooks/depcruiser-migrations/manifests",
    "editTools": ["Edit", "Write", "MultiEdit", "NotebookEdit", "apply_patch"]
  }
}
```

Also update `scripts[id=skill-indexer].settings.scanRoots` if this skill should be auto-indexed:

```json
{ "path": "E:/hooks/depcruiser-migrations", "source": "hooks", "scope": "all" }
```

Do **not** put active migration state in `config.json`. Active migration state belongs in:

```text
E:/hooks/.state/depcruiser-migrations/active.json
```
