# Migration · Repo structure

{/* Hand-authored design proposal — NOT generated from contracts. Status: drafting. Layout/naming decisions here are intended to be fed back and locked into the repo-process and backend-api contracts. */}

The target repository layout for **kai-chattr** and the manifest for moving the
legacy **chattr** codebase into it. See the [Overview](/migration) for the
fusion thesis and the migration method this page operates within.

> **Status: drafting.** The `repo-process` and `backend-api` contracts leave
> layout and naming as open items. This proposal defines them so they can be
> reviewed and locked back into those contracts.

## Principles

- **Copy-only — legacy is never touched.** Every file lands in kai-chattr by
  copy. The legacy repo keeps operating from its original location with its
  original files. No moves, no deletions, no edits to the legacy tree.
- **Outside-in.** The governed shell (contracts · enforcement · docs) is
  completed first; application code is brought in afterward, one slice at a
  time, each slice conforming to its contract.
- **Reorganize the backend, rebuild the frontend.** The Python backend is
  genuinely portable and is copied into a clean package. The legacy `static/`
  frontend is **reference only** — the [frontend contract](/contracts/frontend)
  requires a v3 rebuild on the adopted stack.
- **Contracts are the source of truth.** Python via `uv` only, no repo-local
  `.venv`, dependencies allowlisted, secrets via SOPS — see
  [repo-process](/contracts/repo-process).

## Target layout

```text
kai-chattr/
├─ apps/                          # deployable end-user apps
│  ├─ web/                        # v3 UI — OpenHands stack (React 19 · RR7 · Vite · Tailwind v4 · TanStack · Zustand · Monaco), reskinned to v3/KAI
│  ├─ desktop/                    # RESERVED — local Tauri build, scoped later
│  └─ devdocs/                    # this fumadocs site
├─ services/
│  ├─ api/                        # FastAPI chat server (Python, uv) — package `app`, dist `kai-chattr-api`
│  │  ├─ app/
│  │  │  ├─ core/                 # store · registry · router · rules · summaries · archive · jobs
│  │  │  ├─ sessions/             # session_engine · session_store · schedules
│  │  │  ├─ agents/               # agents
│  │  │  ├─ mcp/                  # mcp_bridge · mcp_proxy
│  │  │  ├─ api/                  # routes — system unprefixed + product under /v1
│  │  │  ├─ events/               # jsonl_stream · terminal_event_schema
│  │  │  ├─ launch/               # chattr_launcher + agents.toml
│  │  │  ├─ wrappers/             # wrapper · wrapper_api · wrapper_unix · wrapper_windows
│  │  │  ├─ settings.py           # pydantic-settings (from config_loader)
│  │  │  └─ main.py               # app factory + entry (from app.py / run.py)
│  │  ├─ pyproject.toml  uv.lock  # uv; deps allowlisted
│  │  ├─ Dockerfile  fly.toml     # container + Fly deploy surface (fly.toml reserved)
│  │  └─ tests/                   # pytest suite
│  └─ memory/                     # RESERVED — future extracted-Letta substrate (cross-repo-shareable), NOT inside api/
├─ packages/
│  └─ schemas/                    # canonical JSON Schemas + session_templates — Python reads BY PATH (jsonschema); TS imports the same JSON. Not copied, not codegen'd.
├─ scripts/launchers/             # windows/*.bat + unix/*.sh agent launchers
├─ governance/  secrets/  docs/   # governance · SOPS · working docs
└─ package.json + pnpm-workspace.yaml   # plain pnpm-workspaces (NO Turborepo)
```

The `apps/* · services/* · packages/*` split is the standard polyglot-monorepo
pattern on **plain pnpm-workspaces** (no Turborepo — a future explicit decision,
not a default). The Python service is a single installable package (`app`, dist
`kai-chattr-api`) mirroring `blockdata/services/api`, with its own PEP-621
`pyproject.toml`.

## Copy manifest

### `services/api/` — Python backend (copy + repackage)

| Legacy | New — `services/api/src/chattr_api/` |
|---|---|
| `store · registry · router · rules · summaries · archive · jobs` | `core/` |
| `session_engine · session_store · schedules` | `sessions/` |
| `agents` | `agents/` |
| `mcp_bridge · mcp_proxy` | `mcp/` |
| `app.py` + `server/api/{platform,terminal,desktop_runtime}` | `api/` (`app.py` → `main.py` factory) |
| `server/events/*` | `events/` |
| `server/launch/{chattr_launcher,agents.toml}` | `launch/` |
| `wrapper · wrapper_api · wrapper_unix · wrapper_windows` | `wrappers/` |
| `config_loader` | `config.py` |
| `run.py` | `__main__.py` / `[project.scripts]` |
| `config.toml`, `config.local.toml.example` | templates (not `config.local.toml`) |
| `pyproject.toml` + `requirements.txt` | merged → `pyproject.toml` (lock regenerated; deps must pass the allowlist) |
| `tests/*` + `conftest.py` | `services/api/tests/` |

### `packages/schemas/` — shared (copy)

`schemas/*.json` (`session-draft-message-metadata`, `session-start-request`,
`session-template`) + `session_templates/*.json` (`code-review`, `debate`,
`design-critique`, `planning`).

### `apps/web/` — frontend (rebuild on the adopted stack; copy assets only)

- **Stack:** the OpenHands frontend stack (see [Overview](/migration)) — adopt
  it so OH's chat surface, tabbed RightDock tool-dock, xterm Terminal tab, and
  Monaco Code tab are **adapt-and-reskin**, not rebuild-from-zero.
- **Copy → `apps/web/public/`:** `sounds/*`, `logo.png`, `favicon.ico`.
- **Reconcile (rebuild input):** legacy `design-system.css` tokens → v3/KAI
  theme; the v3 standalone specimen is the design canon.
- **Not copied:** every `static/*.{js,css,html}` and `mocks/` — reimplemented as
  components per the frontend contract.

### `scripts/launchers/` (copy)

`windows/*.bat` + `uv-env.bat` → `windows/`; `macos-linux/*.sh` → `unix/`.
Internal paths updated for the new layout.

### New wiring (authored, not copied)

Root `package.json` workspaces + `pnpm-workspace.yaml` (**plain pnpm-workspaces, no Turborepo**);
`services/api/pyproject.toml` under `uv`. `UV_PROJECT_ENVIRONMENT` resolves by
context: an env **outside the checkout** locally, an **ephemeral uv env** in CI,
and **the container itself** on Fly.

## Excluded / deferred

- **Runtime state:** `data/`, `logs/` (gitignored — not migrated).
- **Dev scratch:** `_diag_imports.py`, `build_release.py`, `__pycache__/`.
- **Desktop (open):** `apps/desktop/` is reserved. The legacy `desktop/` is a
  Phase-1 Tauri scaffold (PTY/xterm not wired) whose React pages are borrowed,
  incomplete IDT donations on conflicting tokens; its one durable asset is the
  native PTY/OSC/Job-Object layer (`src-tauri`, borrowed from Terax, Apache-2.0,
  attribution must travel). **What to bring over is a later decision** — the
  local desktop build is out of scope for now.
- **Backend desktop-runtime host:** `server/api/desktop_runtime.py` + events +
  its contract tests are part of `services/api` and travel regardless of the
  Tauri-client decision.

## Suggested build order

0. **Gate — contract-encode (governance-first, BLOCKING).** Encode this layout +
   naming into `repo-process.json` (imports/path conventions, file & directory
   naming) and `backend-api.json` (router layout, error envelope); port
   `check-contracts.mjs` (kai-chattr lacks it) + `governance/schemas/`; run
   `check-contracts` **GREEN before any code is copied**. This is the
   decide→encode→check→code discipline; no code-before-contract.
1. Workspace wiring + reserved dirs (`apps/desktop/`, `packages/schemas/`, `services/memory/`).
2. `services/api` skeleton — `pyproject.toml`, `app/` package dirs, `uv`
   (see `docs/plans/0001-services-api-skeleton-plan.md`).
3. Copy backend modules per the manifest; fix imports to `app.<domain>.*`.
   **Top risk — the import-rewrite is the highest-breakage step:** gate it on the
   suite going green under `uv`, and **lock the count (22 `test_*.py` modules +
   `conftest.py`)** to catch dropped coverage.
4. Copy shared schemas (consumed by path) + launcher scripts (path-corrected).
5. `apps/web` scaffold on the adopted stack + asset copy (the UI rebuild is a
   later frontend slice).
