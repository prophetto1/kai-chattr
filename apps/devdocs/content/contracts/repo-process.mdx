# Repo & process contract

{/* GENERATED from governance/contracts/repo-process.json by governance/scripts/build-contract-docs.mjs — do not edit by hand. */}

Repo hygiene and process: secrets via SOPS only (no plaintext env), the Python toolchain (uv), import/path conventions, file & directory naming, and commit conventions. The dependency allowlist is its own contract — see Dependencies.

## The rule

1. 🔒 Secrets go through SOPS only. No plaintext `.env` files anywhere, under any name. A `.gitignore` is NOT protection. Docs reference SOPS pointers + commands, never raw values.
2. 🔒 `uv` is the only Python package manager/toolchain for the backend. No repo-local `.venv` (cross-drive hardlink failures) — `UV_PROJECT_ENVIRONMENT` points uv at an environment outside the checkout for LOCAL dev; in CI it is an ephemeral uv env, and inside a container the container itself is the env. Python dependencies are declared in `pyproject.toml` and allowlisted in `governance/allowed-deps.json`.
3. 🔒 Backend imports use the installed package root `app`: modules import as `app.<domain>.*` (e.g. `app.core.store`, `app.api.system`). No `src/`-layout, no `sys.path` hacks, no relative imports across domain packages.
4. 🔒 Monorepo layout `apps/<app>/`, `services/<service>/`, `packages/<pkg>/` on plain pnpm-workspaces (NO Turborepo). `services/api` is one installable package named `app` (distribution `kai-chattr-api`) with domain subpackages `core/ sessions/ agents/ mcp/ events/ launch/ wrappers/ api/`. Python modules are snake_case; JS workspace packages are kebab-case.
